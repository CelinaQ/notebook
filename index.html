Hopes & Dreams

As a photographer, I work with Photoshop all day long. I often use a script to create files used for my company's website. I didn't write the script that I use, and if something needs to be changed, I have to ask the company developer to do it. I often have to wait for his response and go back and forth several times to ensure that we are on the same page about what is needed. As someone who values efficiency, I'd like to be able to tweak the script if necessary without a bunch of back and forth.

I flourish in life when I'm learning new things. I've had a camera in my hand since I was 4 years old. The bucket of new knowledge from which I can draw in relation to photography is very small. I'm very proficient with Photoshop, I have extensive experience with studio and set lighting, and I know how to create and analyze images.

With web development, the bucket of knowledge from which to draw is the size of an ocean for me. It's unexplored territory in my life and that is incredibly exciting. By the end of this class, I want to be able to create websites for clients. I also want to know where to look when I need answers to my questions as I take on more complex challenges. I would also like to establish good working relationships with my classmates so that we are able to assist each other professionally in the future.

Lastly, I am excited to use my new knowledge as a developer to increase my income. I'm not going to quit my day job anytime soon, but it will be fun to take on side projects to grow my skillset and gain experience while bringing in some extra income. My plan is to continue working my salaried position with all the benefits that entails, and then see where this new (to me) knowledge base leads.

Every card has a title.

This card has a paragraph of text with a link. It also has a second sentence of text with a link.

A card is a simple unit of information. We'll use the < article> tag to represent a card and then style it with CSS.

Every card has a width.

But we never use pixels (px) to determine the size of a card.

We use measurements that flex, like percents (%) or that relate to the size of the content like root "em" units (rem), which relate the size of the elment to the text size and thus make the card very readable.

This card has lists.

Lists are everywhere in web applications. They are used for navigation (a list of links):

  • Go Here
  • Go There
  • Go Someplace Else

Or as a way to display sets of things

  • Red
  • Blue
  • Green

You will use lists a lot.

Glossary

This card collects terms and their definitions into a Description List Element (dl) for later reference.

tag name

  • The word inside the HTML tag that defines the element

    • tag

  • A tag name and opening and closing delimiters. A closing tag puts a forward slash in front of of the tag name.

    • element

  • An opening tag, some content, and a closing tag. Some elements don't have closing tags and some are self closing.
